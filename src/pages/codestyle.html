Очевидно, охватить все ситуации слишком сложная задача даже с помощью абстрактных правил, что уж говорить про конкретные. Поэтому, договоримся считать, что все что не охватывается данными тут правилами не подлежит стандартизации, и может быть любым; а соблюдение всех этих правил, означает придерживание этого стиля.

(0) Отступы. (1) пробелы. (2) скобки. (3) идентификаторы. (4) переменные. (5) комментарии. (6) блоки if-else. (7) проверка if-else. (8) в указателях *. (9) тернарный оператор. (10) бесконечный цикл. (11) указание типа. (12) инкремент-декремент. (13) выравнивание. (14) switch. (15) массивы. (16) goto. (17) препроцессор. (18) типы. (19) iool. (20) возврат функции. (21) функции нужные в одном файле. (22) перменные n. (23) временные макрос функции. (24) выделение памяти. (25) typedef. (26) приведение функций к void. (27) комментарий NOTREACHED. (28) символ _ в идентификаторах.

1. Общее


(0) Отступы. Используются \t в 8 пробелов; (1) пробелы. В выражениях все пробелы опускаются, например, вместо [10 + 1 / 2], будет [10+1/2]. (2) скобки. Если это возможно, не указывать скобки {}, в ином случае, если скобки {} не находятся ни в никаких других {}, 

       они таковы,
		int main(void)
		{
			...,
		}
       в ином случае,
		if (!0&&1||4) {
			...,
		}
(3) идентификаторы. Их стиль это flatcase, иногда можно snake_case. (4) переменные. Все переменные следует распологать в самом начале функции, и объявлять перед их использованием, чем длиннее объявление переменной, тем выше других она должна быть. Например, 
     не так,
		static int sum(int x, int y)
		{
			int res=x+y;
			const char *msg="hello\n";
			printf("%s",msg);
			return res;
		}
     а вот так,
		static int sum(int x, int y)
		{
			const char *msg;
			int res;

			res=x+y;
			msg="hello\n";
			printf("%s",msg);

			return res;
		}
(5) комментарии. Комментарии следует оставлять всегда через /* */. И лучше 
     в таком стиле,
		/*
		 * my
		 * comment
		 */
(6) блоки if-else. Следует объявлять 
      не вот так,
		if (!k) {
			...,
		} else {
			...,
		}
      а так,
		if (!k) {
			...,
		}
		else {
			...,
		}
(7) проверка if-else. Проверку на NULL следует делать, 
	вот так,	а не так,
	if (!name)	if (name==NULL)
	if (name)	if (name!=NULL)
И вообще такая проверка предпочтительнее других, однако с числами надо быть аккуратнее. (8) в указателях *. В указателях следует ставить * вот так, char *name, а не так, char* name. (9) тернарный оператор. Оператор ()?():(), следует использовать часто. (10) бесконечный цикл. Предпочительнее использовать for (;;), или цикл через goto. (11) указание типа. Тип следует указывать на той же строчке, на которой и идентификатор. 
	const char *n;
	int main(void)
(12) инкремент-декремент. В случаях когда префексный и постфиксный инкремент-декремент равносильны, предпочительно использовать, префиксный (++res, --ret). (13) выравнивание. Выравнивания через \t, такого или похожего типа, приветствуются, 
	int		n;
	const char	*txt;
	short		j;
(14) switch. Switch всегда предпочительнее if-else. (15) массивы. Лучше вместо char *argv[], писать char **argv, и т.д.


2. Остальное


(16) goto. Ограничений на использование нету; предпочтельные названия меток, таковы, L0-Ln, err, exit, write, read, out, warn. (17) препроцессор. #define следует использовать как можно чаще. (18) типы. Для unsigned типов, следует использовать эти названия, u8, u16, u32, u64, либо же, u_char, u_short, u_int, u_long. (19) bool. Bool допустимо использовать, однако лучше u_char. А ставить 

     его в 1 лучше так,
		++flag;
     ставить в 0 так,
		--flag;
(20) возврат функции. Возврат функций (особенно чтения-записи) когда это возможно, следует принимать в переменную с названием ret. А возращать, когда это возможно, в перменной res. (ret=return, res=result). (21) функции нужные в одном файле. Если функция используется только в одном файле, лучше объявить ее через static. (22) перменные n. Переменные с названием n, n0, ..., следует использовать при вычислениях, или даже в качестве счетчиков. (23) временные макрос функции. Временные макрос функции предпочтительны, главное в конце вызвать #undef. (24) выделение памяти. Предпочтительнее вместо malloc(100) писать calloc(1,100), и вообще лучше использовать calloc. (25) typedef. Использовать свои typedef приветствуется, особенно для структур в таком стиле, [structname]_t. (26) приведение функций к void. Это допустимо, но необязательно, 
	(void)numerus();
(27) комментарий NOTREACHED. Если функция не может прийти к последней строчке, то на эту строчку лучше поставить следующий комментарий, 
	static int via(void)
	{
		for (;;)
			return 0;
		/* NOTREACHED */
	}
(28) символ _ в идентификаторах. Символ _ в идентификаторах можно использовать. Особенно когда нужно что то зарезервировать или скрыть. 
	int __tmp;
	const char *_name;
	struct __ratio__
	{
		...,
	};
